# Processed Interpolation Outputs

This folder is generated by:

```bash
cd analysis/toronto_election_turnout/interpolation
python3 run_interpolation.py
```

The workflow writes one set of files per election plus combined audit,
exclusion, and validation reports. The target universe is the 585 Toronto
2021 CT polygons linked to Toronto DAs.

Mapped polling divisions and geometryless district records both use DA
`citizen_canadian_18over` weights. CT results include official-elector-based
diagnostic turnout plus `estimated_turnout_citizen_18plus` /
`estimated_participation_citizen_18plus`.

## Election-Specific Outputs

Each election prefix has:

- `*_poll_to_ct_crosswalk.csv`
- `*_district_to_ct_crosswalk.csv`
- `*_ct_estimated_results.csv`
- `*_ct_candidate_estimated_votes.csv`
- `*_excluded_unallocated.csv`
- `*_no_geometry_allocation_audit.csv`
- `*_turnout_comparison.csv`
- `*_audit.csv`
- `*_validation.csv`
- `*_summary.json`

## Combined Outputs

- `AUDIT_FINDINGS.md`
- `census_input_audit.json`
- `suppressed_da_audit.csv`
- `interpolation_audit.csv`
- `excluded_unallocated_report.csv`
- `no_geometry_allocation_audit.csv`
- `turnout_comparison.csv`
- `validation_report.csv`
- `validation_summary.json`

`validation_summary.json` retains the interpolation runner's original
turnout-proximity diagnostic. Its older `map_gate_passed` field can be false
even when all vote-integrity checks pass, because contextual published turnout
rates can use different geographies and denominators.

Coverage and temporal diagnostic outputs are generated separately by:

```bash
cd analysis/toronto_election_turnout/interpolation
python3 audit_geographic_temporal_coverage.py
```

They include:

- `geographic_coverage_audit.csv`: checks whether selected election districts
  require additional CT/DA target geography beyond the stored Toronto universe.
- `temporal_population_proxy_audit.csv`: reports official 2021-to-election-year
  total-population and age-18+ proxy changes, but does not apply them to
  production citizen-18+ turnout.
- `geographic_temporal_audit_summary.json`: summarizes the decisions.

Official election-result reconciliation outputs include:

- `official_party_vote_reconciliation.csv`: official, normalized, and
  interpolated party totals globally and by ward/riding.
- `official_candidate_vote_reconciliation.csv`: official and normalized
  candidate totals globally and by ward/riding.
- `official_result_reconciliation_summary.json`: overall reconciliation gate.
- `turnout_reference_audit.csv`: labels turnout references by geography,
  status, and suitability for direct comparison.

Validated map-ready GeoJSON is written to the adjacent `map/` directory by
`analysis/toronto_election_turnout/interpolation/build_map_data.py`.
`map/map_build_summary.json` is the authoritative map-build record. It requires
primary vote preservation, complete no-geometry vote allocation, no
vote-bearing exclusions, and official result reconciliation.
